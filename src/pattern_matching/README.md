#   

Rust 中的 match 是一种强大的 模式匹配（pattern matching） 控制流结构，可以用来分支执行逻辑，类似于其他语言中的
switch，但功能更强大、更安全。

**match 特性**

✅ 穷尽性检查（exhaustiveness checking）
Rust 编译器强制要求 match 分支覆盖所有可能的情况，否则编译失败。这提高了程序的健壮性和安全性。

✅ 模式解构
支持结构体、元组、枚举、引用等类型的复杂模式解构。

✅ 支持守卫（guard）
可以在分支中加入条件判断

| 特性      | 说明                                  |
|---------|-------------------------------------|
| 表达能力强   | 支持复杂模式匹配和条件                         |
| 编译期检查   | 保证所有情况都被覆盖                          |
| 与枚举紧密结合 | 特别适合 `Option`, `Result`, 自定义 `enum` |
| 解构能力    | 能直接从结构体/元组中提取值                      |
| 安全性高    | 强类型、不可遗漏的分支避免潜在逻辑漏洞                 |

| 特性    | `match`          | `if let`   |
|-------|------------------|------------|
| 表达能力  | 强，支持多个模式、守卫、解构等  | 简洁，仅适合一个模式 |
| 编译期检查 | 全面，必须穷尽所有分支      | 不强制处理其他情况  |
| 推荐场景  | 多种可能情况、复杂结构、需要解构 | 只关心一个模式时使用 |
